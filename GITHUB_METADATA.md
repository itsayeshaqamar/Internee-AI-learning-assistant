## 💬 GitHub Discussions Setup

If you choose to enable **GitHub Discussions** for community support, configure the following categories:

1. **📢 Announcements:** Major releases, milestones, or project updates from maintainers.
2. **💡 Ideas:** Suggesting new features, user stories, or design feedback.
3. **🙋 Q&A:** Asking for help with local setup, installation, or configuration.
4. **💬 General:** General conversations, learning resources, and networking among Internee.pk interns.
5. **🐛 Bug Reports:** Reporting edge case issues and confirming troubleshooting steps.

---

## 🛠️ Repository Feature Settings

In the **General** settings tab, configure the following features:

- **Wikis:** Disable (all documentation is self-contained in markdown files within the repo for easier versioning).
- **Sponsorships:** Optional (enable if you want to support open-source development).
- **Projects:** Enable (useful for tracking features using Kanban boards).
- **Releases:** Enable (creates release milestones associated with `CHANGELOG.md` updates).
- **Pull Requests:**
  - Enable **Allow merge commits**.
  - Enable **Allow squash merging** (recommended for clean, linear commit histories on main).
  - Enable **Automatically delete head branches** (keeps repository branch listing clean after merging).
